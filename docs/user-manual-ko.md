# 마케팅 리서치 컴패니언 사용자 매뉴얼

## 1. 이 서비스로 할 수 있는 일

마케팅 리서치 컴패니언은 사용자의 ChatGPT/Codex 계정으로 다음 업무를 수행하는 개인용 로컬 웹 서비스입니다.

- 시장 동향과 경쟁 제품 조사
- 고객 리뷰와 VOC의 만족·불만·반복 키워드 분석
- 경쟁 제품별 가격, 판매 근거와 예상 매출 산식 정리
- 구매 고객 페르소나 정의
- 자사 상품 스펙, 가격 포지셔닝과 런칭 테스트 제안
- 경쟁 제품별 공식 판매 이미지 기본 수집
- 선택 시 공식 이미지를 참조한 동일 제품 사용 장면 생성
- 대표 보고서 생성과 PDF 저장

서비스 제공자의 공용 API 키를 사용하는 SaaS가 아닙니다. 사용자마다 자신의 컴퓨터에 설치하고 자신의 ChatGPT 계정을 연결합니다.

## 2. 준비 사항

- macOS 또는 Linux 컴퓨터
- Node.js 20 이상
- ChatGPT 데스크톱 앱에 포함된 Codex 또는 Codex CLI
- Codex를 사용할 수 있는 ChatGPT 계정

macOS에서는 ChatGPT 데스크톱 앱에 포함된 Codex를 자동으로 찾습니다. Windows에서는 자동 시작 서비스가 지원되지 않지만 `npm start`로 직접 실행할 수 있습니다.

## 3. 가장 쉬운 설치 방법

시스템 요구사항 설치까지 Codex에게 맡기려면
[Codex용 설치·온보딩 전체 지시문](install-onboarding-order-ko.md)을 그대로 전달합니다.
간단히 요청할 때는 아래 내용을 사용합니다.

```text
https://github.com/NineTigers/marketing-research-companion

현재 운영체제에서 Git, Node.js 20 이상과 Codex 요구사항을 먼저 확인해줘.
누락된 시스템 도구가 있으면 설치 계획을 설명하고 내 승인을 받은 뒤 설치해줘.
그 다음 이 저장소를 설치하고 doctor, check, test를 통과시켜줘.
서비스를 시작한 뒤 내 ChatGPT 계정 상태와 실제 접속 URL을 알려줘.
```

Codex는 이미 준비된 도구를 다시 설치하지 않습니다. 관리자 권한이나 시스템 변경이 필요한
항목은 먼저 설치 계획을 보여주고 사용자의 승인을 받아야 합니다. 비밀번호, ChatGPT 토큰,
API 키를 대화창에 입력해서는 안 됩니다.

Codex가 설치를 마치면 `http://127.0.0.1:8787` 형태의 주소를 알려줍니다. `8787`이 이미 사용 중이면 `8788`처럼 다음 사용 가능한 포트가 자동으로 선택됩니다. 반드시 Codex나 터미널이 알려준 실제 주소를 여세요.

## 4. 직접 설치하는 방법

터미널에서 다음 명령을 순서대로 실행합니다.

```bash
git clone https://github.com/NineTigers/marketing-research-companion.git
cd marketing-research-companion
npm run doctor
npm start
```

`npm run doctor` 결과가 모두 `OK`이면 준비된 상태입니다. `npm start`가 출력한 접속 주소를 브라우저에서 엽니다. 터미널을 종료하면 서비스도 종료됩니다.

## 5. ChatGPT 계정 연결

### 이미 Codex에 로그인한 경우

상단에 **내 ChatGPT 계정으로 조사할 수 있습니다**와 계정 이메일이 표시됩니다. 별도 로그인이 필요하지 않습니다.

### 로그인이 필요한 경우

1. **ChatGPT로 연결**을 누릅니다.
2. 열린 브라우저에서 본인의 ChatGPT 계정으로 로그인합니다.
3. 원래 화면이 **사용자 Codex 준비** 상태로 바뀔 때까지 기다립니다.
4. 연결 계정이 본인 계정인지 이메일과 플랜을 확인합니다.

이 앱은 ChatGPT 토큰을 읽거나 저장하지 않습니다. 인증 정보는 Codex가 관리합니다.

> **연결 해제 주의:** 연결 해제는 같은 기기의 Codex CLI나 다른 Codex 화면에도 영향을 줄 수 있습니다.

## 6. 첫 조사 실행

### 필수 입력

1. **제품 또는 제품군**: 조사할 대상을 구체적으로 입력합니다. 예: `24~47개월 어린이집 낮잠용 토들러 베개`
2. **고객 단계**: 실제 사용 연령이나 구매자 유형을 선택합니다.
3. **요청 업무**: 시장 분석, VOC 기반 상품 개발, 소싱, 유통 검토 등 목적을 선택합니다.
4. **필요한 결정**: 보고서를 보고 최종적으로 결정할 내용을 선택합니다.

### 결과 품질을 높이는 입력

- **배경과 특히 확인할 점**: 직접 경쟁 제품의 범위와 제외 대상을 적습니다.
- **고객 리뷰·VOC 원문**: 상담 기록이나 리뷰 문장을 그대로 입력합니다.
- **우선 확인할 출처 URL**: 공식몰이나 판매 페이지를 한 줄에 하나씩 입력합니다.
- **시장 범위**: 대한민국 우선, 국내 전용, 글로벌 중 하나를 선택합니다.
- **조사 범위**: 빠른 검토, 표준 조사, 심층 조사 중 선택합니다. 모든 범위에서 `gpt-5.6-terra / high`를 우선 사용하고 Terra 미지원 계정에서만 `gpt-5.5 / high`를 사용합니다.
- **공식 제품 이미지**: 선택 여부와 관계없이 경쟁 제품별 공식 판매 페이지에서 수집합니다.
- **경쟁 제품별 동일 제품 사용 장면 생성**: 기본값은 선택하지 않음입니다. 체크하면 공식 제품 이미지를 참조해 각 제품의 실제 사용 맥락 이미지를 만들고 공식 이미지 옆에 배치합니다. 현재 계정 런타임이 이미지 생성을 지원하지 않으면 생성만 선택할 수 없습니다.

좋은 입력 예시는 다음과 같습니다.

```text
제품: 토들러 낮잠 베개 단품
고객: 24~47개월 어린이집 낮잠 준비 가정
결정: 샘플 제작 여부
확인할 점: 낮잠이불과 매트는 직접 경쟁 제품에서 제외하고 베개 단품만 비교.
가격, 실제 판매 링크, 반복 VOC, 제품별 예상 매출 산식과 출처를 포함.
```

입력이 끝나면 **조사 실행**을 누릅니다.

## 7. 조사 진행과 결과 확인

**조사 진행** 영역에서 다음 상태를 확인할 수 있습니다.

- `대기`: 실행 준비 중
- `조사 중`: 사용자 Codex가 웹 근거를 수집·분석 중
- `완료`: 보고서 생성 완료
- `확인 후 제출`: 보고서는 완성됐지만 안전·근거 확인 항목이 있음
- `실패` 또는 `중단됨`: 원인을 확인한 뒤 다시 실행 가능

실행 중에는 **조사 중단**을 사용할 수 있습니다. 실패하거나 중단된 업무는 **다시 실행**할 수 있습니다.

## 8. 대표 보고서 읽는 방법

보고서는 다음 순서로 구성됩니다.

1. 마케팅팀 제안과 대표 승인 요청
2. 선택한 업무에 맞는 추천·소싱·유통·검증 결론
3. 선택한 근거별 수치 차트
4. 시장 신호와 사업적 의미
5. 공식 이미지가 연결된 실제 경쟁 제품, 판매 URL, 가격과 검산된 매출 추정
6. 표본수·언급수·비중이 표시된 VOC 만족·불만과 반복 고객 언어
7. 성공 원인 분석
8. 구매 페르소나
9. 자사 상품 스펙과 가격 제안
10. 런칭 테스트, 리스크와 다음 행동
11. 확인한 근거 목록

매출 추정은 실제 매출액이 아닐 수 있습니다. 제품별 계산 근거는 `공식 판매량·공개 매출 → 구매·주문 수 → 리뷰 수÷리뷰 작성률 → 조회 수×구매 전환율 → 댓글·좋아요·저장 신호` 순으로 선택합니다. 공식 가격이 없으면 판매 페이지 표시 가격, 동일 SKU 가격, 동일 채널 가격 범위를 차례로 사용합니다. 보고서의 적용 방식, 기간, 산식, 입력값, 가격 근거, 전환 가정, 보수·기준·낙관 범위와 한계를 함께 확인하세요.

### PDF로 저장

보고서 상단의 **PDF 저장**을 누른 뒤 브라우저 인쇄 화면에서 **PDF로 저장**을 선택합니다.

## 9. 차트 요청과 지시 보관

- **차트 요청**: VOC, 반복 키워드, 매출 시나리오, 경쟁 제품 포지셔닝 등에 사용할 차트 형식을 저장하고 다음 조사 요청과 보고서에 반영합니다. 직접 입력해 저장한 사용자 지정 근거도 함께 전달됩니다.
- **지시 보관**: 아직 조사하지 않은 업무 지시와 상태를 보관합니다.
- **대표 보고서**: 이 설치본에서 생성한 보고서를 다시 엽니다.

업무 지시와 차트 선택은 서버와 브라우저에 동기화됩니다. 조사 결과와 보고서는 설치 폴더의 `.data`에 저장됩니다.

## 10. 항상 실행되게 설정

macOS와 Linux에서는 다음 명령으로 로그인 후 자동 시작을 설정할 수 있습니다.

```bash
npm run service:install
npm run service:status
```

`service:status` 마지막에 표시되는 **서비스 URL**을 사용합니다.

자동 시작을 제거하려면 다음을 실행합니다.

```bash
npm run service:uninstall
```

## 11. 업데이트

### 웹에서 업데이트

1. 왼쪽 **운영 참고**를 열고 **서비스 관리**로 이동합니다.
2. **업데이트 확인**을 누릅니다.
3. 새 버전이 있고 자동 적용 가능 상태이면 **업데이트**를 누릅니다.
4. 자동 시작 서비스는 적용 후 재시작되며 화면이 자동으로 갱신됩니다.

웹 업데이트는 공식 저장소의 `main` 브랜치, 로컬 수정 없음, fast-forward 가능 상태에서만
작동합니다. 실행 중인 조사가 있거나 로컬 파일을 수정했거나 Git 이력이 갈라졌다면 데이터를
덮어쓰지 않고 적용을 중단합니다.

### 터미널에서 업데이트

설치 폴더에서 다음 명령을 실행합니다.

```bash
git pull --ff-only
npm run doctor
npm run check
```

자동 시작 서비스를 사용한다면 업데이트 후 서비스를 다시 설치하면 현재 설치 경로와 실행 파일 정보가 갱신됩니다.

```bash
npm run service:install
```

업데이트 전에 중요한 보고서는 PDF로 저장하거나 `.data` 폴더를 백업하세요.

## 12. 데이터 관리

- 조사 지시, VOC, 보고서: 설치 폴더의 `.data`
- ChatGPT 인증 정보: Codex가 관리하며 `.data`에는 저장하지 않음
- 보고서 한 건 삭제: 웹의 조사 기록에서 **삭제** 사용
- 전체 로컬 데이터 삭제: 서비스를 중지하고 `.data` 폴더 삭제

VOC나 상품 계획에 개인정보와 기밀 정보가 포함될 수 있습니다. 승인된 보관 정책 없이 민감 정보를 입력하지 마세요.

## 13. 다른 기기에서 보기

서비스를 인터넷에 직접 공개하지 마세요. 다른 기기에서 확인해야 할 때는 인증된 SSH 포트 포워딩을 사용합니다.

```bash
ssh -L 8787:127.0.0.1:8787 <사용자>@<설치된-컴퓨터>
```

서비스가 다른 포트를 사용 중이면 양쪽 포트 번호를 실제 포트로 바꿉니다.

## 14. 문제 해결

### `Codex CLI를 찾을 수 없습니다`

- macOS: ChatGPT 데스크톱 앱이 `/Applications`에 설치되어 있는지 확인합니다.
- 별도 Codex CLI를 사용한다면 `.env`의 `CODEX_BIN`에 실행 파일 경로를 지정합니다.

### `ChatGPT 계정 로그인이 필요합니다`

웹에서 **ChatGPT로 연결**을 누르거나 터미널에서 `codex login`을 실행합니다.

### 기본 주소로 연결되지 않습니다

터미널의 `Marketing research team:` 다음 주소를 확인합니다. 자동 시작 중이면 다음을 실행합니다.

```bash
npm run service:status
```

### 조사가 오래 걸립니다

실제 웹 검색과 근거 확인이 포함되므로 몇 분이 걸릴 수 있습니다. 빠른 판단은 **빠른 검토**, 중요한 상품 결정은 **표준 조사** 또는 **심층 조사**를 사용합니다.

조사 범위는 확인하는 근거와 대안의 폭을 조절합니다. 빠른 검토를 선택해도 모델이나 추론 강도는 낮아지지 않습니다.

Terra 대신 GPT-5.5가 적용되면 상단 상태에 **GPT-5.5 · HIGH**와 대체 적용 이유가 표시됩니다. 두 모델 모두 지원되지 않으면 조사를 시작할 수 없습니다.

### 보고서가 `확인 후 제출` 상태입니다

실패가 아니라 추가 검토가 필요한 상태입니다. 보고서의 **제출 전 확인할 항목**과 출처를 확인하세요.

### 화면만 확인하고 싶습니다

실제 웹 조사 없이 데모 모드로 실행할 수 있습니다.

```bash
MARKETING_RUNTIME=demo npm start
```

데모 결과를 실제 시장 사실이나 사업 결정의 근거로 사용하면 안 됩니다.
