# 정식 프로젝트 전환 안내

이 문서는 과거 Workbench의 `project/asset/marketing-presentation` 또는 그 아래 `dist`
실행본을 사용하던 설치를 독립 프로젝트 체크아웃으로 전환할 때 사용합니다.

## 원칙

- 기존 설치본과 데이터는 확인 전 삭제하지 않습니다.
- 새 소스는 공개 저장소를 `projects/marketing-research-companion` 또는 사용자가 선택한 별도
  디렉터리에 clone한 체크아웃을 사용합니다.
- 조사 이력은 운영체제 사용자 데이터 디렉터리로 옮긴 뒤 서비스가 그 위치를 사용하게 합니다.
- 한 번에 두 서비스가 같은 포트나 같은 데이터 디렉터리를 쓰지 않게 합니다.

## 전환 순서

1. 기존 서비스의 `npm run service:status`와 `.env`의 `DATA_DIR`을 확인합니다.
2. 기존 서비스를 중지하고 데이터 디렉터리를 백업합니다.
3. 새 프로젝트에서 `npm run doctor`, `npm run check`, `npm test`를 실행합니다.
4. 기존 jobs, reports, assets, UI state와 runtime 파일을 새 사용자 데이터 디렉터리로 복사합니다.
5. 새 프로젝트에서 `npm run service:install`을 실행합니다.
6. 실제 URL, 기존 보고서, 새 데모 작업, 재시작 후 복구를 확인합니다.
7. 검증이 끝난 뒤에만 과거 실행 소스와 백업의 보존 여부를 결정합니다.

데이터 이전은 파일을 덮어쓸 수 있으므로 자동 업데이트가 수행하지 않습니다. 설치 담당자는
원본·대상 경로와 백업을 먼저 제시하고 사용자 승인을 받은 뒤 복사해야 합니다.
