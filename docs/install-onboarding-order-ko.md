# Codex용 설치·온보딩 전체 지시문

아래 지시문을 사용자 자신의 Codex 작업에 그대로 전달합니다. 시스템 프로그램 설치에는 사용자 동의와 운영체제 권한이 필요하므로, Codex가 누락 항목을 먼저 확인하고 한 번의 승인으로 필요한 설치를 진행하도록 구성했습니다.

```text
설치할 공개 저장소:
https://github.com/NineTigers/marketing-research-companion

목표:
이 마케팅 리서치 컴패니언을 내 컴퓨터에 설치하고, 내 ChatGPT/Codex 계정으로
정상 작동하는 로컬 웹 서비스까지 온보딩해줘.

다음 순서로 끝까지 진행해줘.

1. 현재 운영체제, CPU 아키텍처, 사용 가능한 셸과 설치 권한을 확인해줘.
2. 아래 시스템 요구사항을 실제 명령으로 점검해줘.
   - Git
   - Node.js 20 이상
   - Codex CLI 또는 macOS ChatGPT 데스크톱 앱에 포함된 Codex
   - Codex를 사용할 수 있는 ChatGPT 계정
3. 이미 준비된 항목은 다시 설치하지 마.
4. 누락되거나 버전이 낮은 시스템 프로그램이 있으면:
   - 누락 항목, 설치 이유, 사용할 공식 설치 경로 또는 신뢰할 수 있는 OS 패키지 관리자를 먼저 보여줘.
   - 관리자 권한, GUI 설치, 패키지 관리자 설치 또는 시스템 변경이 필요하면 한 번에 묶어 내 승인을 받아줘.
   - 승인 후 설치하고 실제 버전을 다시 확인해줘.
   - 비밀번호, ChatGPT 토큰, API 키를 대화에 입력하라고 요구하지 마.
5. 설치할 위치를 명확히 알려주고 공개 저장소의 main 브랜치를 clone 해줘.
   이미 같은 저장소가 있다면 임의로 삭제하거나 덮어쓰지 말고 상태를 확인한 뒤 안전하게 사용해줘.
6. 저장소 안에서 다음 검사를 실행해줘.
   - npm run doctor
   - npm run check
   - npm test
7. 검사 실패 시 원인을 해결하고 같은 검사를 다시 실행해줘. 테스트를 생략하고 완료라고 하지 마.
8. npm start로 서비스를 시작해줘.
   - 기본 포트 8787이 사용 중이면 앱이 선택한 다음 포트를 확인해줘.
   - 터미널에 출력된 실제 접속 URL을 사용해줘.
9. 웹 서비스의 /api/health 또는 동등한 상태 확인으로 다음을 검증해줘.
   - 서비스 버전
   - Codex 설치 상태
   - 내 ChatGPT 계정 연결 여부
   - 조사 실행 가능 상태
10. ChatGPT 로그인이 필요하면 앱의 Codex 관리형 OAuth 흐름을 열어줘.
    로그인은 내가 브라우저에서 직접 완료하게 하고, 완료 후 연결된 계정을 다시 확인해줘.
11. 서비스가 준비되면 아래 내용을 최종 보고해줘.
    - 설치 위치
    - 설치한 시스템 요구사항과 버전
    - 앱 버전과 Git 커밋
    - 테스트 결과
    - 연결된 ChatGPT 계정 상태
    - 실제 접속 URL
    - 데이터 저장 위치
    - 서비스 중지 및 다시 시작 방법
12. macOS 또는 Linux에서 로그인 후 자동 시작이 유용하면 npm run service:install을 제안해줘.
    자동 시작 설치는 별도 시스템 변경이므로 내 승인을 받은 뒤 진행해줘.

보안 및 범위:
- 앱이나 브라우저 저장소에 ChatGPT 토큰 또는 API 키를 복사하지 마.
- 서비스를 공용 네트워크 인터페이스나 인터넷에 직접 공개하지 마.
- Workbench, 내부 pack 또는 별도 사내 저장소를 설치하지 마.
- 데모 모드 결과를 실제 조사 성공으로 보고하지 마.
- 지원되지 않는 환경이면 가능한 것처럼 우회하지 말고 제한과 다음 행동을 알려줘.
```

## 설치 담당자가 최종적으로 확인할 완료 기준

- 시스템 요구사항 버전이 확인됨
- 공개 저장소의 고정 커밋이 설치됨
- `doctor`, `check`, `test`가 통과함
- 사용자의 ChatGPT/Codex 계정 상태가 확인됨
- 로컬 웹 서비스가 실제 URL에서 응답함
- 데이터 위치와 재시작 방법이 사용자에게 전달됨

시스템 프로그램 설치는 무단으로 수행하지 않습니다. 누락 항목을 발견했을 때 Codex가 설치 계획을 제시하고 사용자가 승인한 범위에서만 진행하는 것이 정상 온보딩입니다.
